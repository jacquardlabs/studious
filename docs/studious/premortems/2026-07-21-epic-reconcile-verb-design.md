# Pre-mortem — `gate-ledger epic-reconcile` (one composite read for work-through's reconcile step)

- Design doc: docs/superpowers/specs/2026-07-21-epic-reconcile-verb-design.md
- Branch: epic/perf-audit-followups--epic-reconcile-verb
- SHA: 5d4a857
- Date: 2026-07-21

| # | Lane | Failure mode | Detection hint |
|---|------|--------------|----------------|
| 1 | technical | The new verb re-derives each story's work-file / gate-file path by hand (independent slugify, independent `--` collapse) instead of calling `cmd_work_get`/`cmd_gate_get`, so the read disagrees with the write side and returns `null` for a story whose file actually exists on disk. | Read `cmd_epic_reconcile` in `bin/gate-ledger`: it must call the three existing read functions, not build a `.studious/.../${slug}--${s}.json` path itself. Add a test where a story has recorded work + gate state and assert `work`/`gate` come back non-null. |
| 2 | technical | `landedButUnmerged` has the wrong sense, or the `git merge-base --is-ancestor` exit-128 case (ref doesn't exist) is folded into `false` instead of "not verified merged", so a genuinely unmerged `landed` story is reported as merged — defeating acceptance criterion 3. | The new test must assert: a truly-merged `landed` story → `false`; a `landed` story never merged → `true`; a `landed` story whose branch ref is missing → `true` (not silently `false`). |
| 3 | technical | A corrupted per-story work/gate file `cat`s fine but the next `jq` parse fails deep in the loop; under `set -uo pipefail` an unchecked command substitution doesn't abort, so a read failure is emitted as a silent `null`/`false` indistinguishable from "legitimately absent" (design Open question 1). | The build must fail closed: a hand-corrupted stored file makes `epic-reconcile` exit non-zero with stderr naming the store and story. Confirm a regression test exercises a corrupted file directly, not just the happy path. |
| 4 | technical | `designDocExists` reads the wrong base worktree for a `landed` story — using the (now-removed) story worktree instead of falling back to `.studious/worktrees/<epic>/__epic` — and reports a false `false`/`null` for a design doc that exists in the epic worktree. | A test with a `landed` story whose design doc lives only in `__epic` must return `designDocExists: true`. Confirm the base is derived from recorded `status`, not read from an untrusted `.worktree` field. |
| 5 | technical | `.epic` is not byte-identical to a bare `epic-get --slug` call (the composite wraps, re-keys, or reshapes it), so `commands/work-through.md`'s "2 · Run the driver script" step feeds `epic-driver.js` an `args.epic` shape it no longer recognizes — a silent contract break at the driver boundary. | Assert `epic-reconcile --slug X | jq -cS .epic` equals `epic-get --slug X | jq -cS .`. In the work-through.md diff, step 2 must read `.epic` out of the reconcile payload, not re-call `epic-get`. |
| 6 | product | `commands/work-through.md`'s "1 · Reconcile" prose adds the new verb but stops acting on `landedButUnmerged` (the flag-and-correct-via-gate-ledger step), so a story whose recorded `landed` isn't on the epic branch is silently trusted and advances as done — the exact failure acceptance criterion 3 exists to prevent, and the trust weakening CLAUDE.md warns against. | The work-through.md diff must keep a branch on `landedButUnmerged: true` that flags and corrects (never by hand). Walk criterion 3 at acceptance: a `landed`-but-unmerged story must still be caught, not passed through. |
| 7 | product | The verb absorbs judgment the design put out of scope — it reads a gate profile and returns each story's `nextPhase`, so phase derivation now lives in `bin/gate-ledger` as well as the prose/script, crossing "code owns bookkeeping; prompts own judgment" and making the ledger the second place that must know how to read a profile. | Grep `cmd_epic_reconcile`: it must emit only the raw + two derived facts in the design's JSON shape, with no `nextPhase` field and no gate-profile read. Any phase-deciding logic in the diff is the finding. |
