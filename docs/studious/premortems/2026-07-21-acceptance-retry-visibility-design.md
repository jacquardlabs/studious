| # | Lane | Failure mode | Detection hint |
|---|------|--------------|----------------|
| 1 | product | The `(resumed)` tag invites but never compels a manual check — a maintainer scanning the report reads `acceptance: SHIP (resumed)` as "resumed, fine" and misses a genuinely slow resumed gate; the tag renders byte-identical for a 5-minute and a 117-minute resume (the design admits this), so the one anomaly this epic exists to surface stays invisible whenever it lands at a resume boundary | Confirm gate-acceptance re-verifies the shipped tag wording reads as "no same-run measurement — a check worth taking," not silent absence; confirm the report asserts no "slow"/"stalled"/"retried" adjective, so the tag invites investigation without claiming a verdict on health |
| 2 | technical | Marker-write is best-effort: a failed Reconcile `work-log --step run-boundary` write leaves the resumed phase with its real prior-run predecessor, so it renders the old misleading cross-boundary number (e.g. `(2877m)`) for that one story — the exact false outlier the HOLD finding named, reintroduced silently on any missed marker | Simulate a failed Reconcile marker write; confirm the render degrades to the known pre-fix behavior for that story alone (a bounded, documented regression) and never crashes the report, and that the design's "best-effort degrade" note survives into the shipped prose |
| 3 | technical | The per-story `work-get` re-read added to the reporting step aborts the whole report block on one missing/corrupt work file instead of skipping just that story | Feed the report run one unreadable/malformed work file; confirm the other stories still render; grep the edit for a per-story best-effort skip, not a single `jq` that aborts on the first bad file |
| 4 | technical | The render rule keys off "predecessor entry's `step` == `run-boundary`"; if the reserved name ever collides with a profile phase, or a marker is non-adjacent to the phase it should tag (dedup/interleaving edge), a real phase renders `(resumed)` or a resumed phase renders a misleading number | Confirm `run-boundary` collides with nothing in `FULL_PROFILE`/`GATES`/`WORKER_PHASES`; test a story with a marker followed by two real same-run phases — only the first should tag, the second should get its accurate number |
| 5 | technical | First-phase duration uses the work file's `createdAt` as predecessor; a pre-existing or hand-edited file missing `createdAt` renders a literal NaN/negative first delta | Run the computation against a work file with no `createdAt`; confirm the first phase shows no duration rather than "NaN"/a negative number |
| 6 | product | Durations render on every phase of every reported story, adding visual weight to the common healthy case and cutting against "lightweight and optional" (PRODUCT.md principle) | Eyeball a normal multi-story landed report; the annotation should be a compact inline parenthetical on the trail, not a separate per-story table |
| 7 | technical | The delta arithmetic ships as prose `jq` in `commands/work-through.md`, outside code, with no `tests/test_gate_ledger.sh` coverage — acceptable only while single-consumer (the design records this as a decision with a promotion trigger) | If `finaleGate` duration surfacing or the `/work-on` takeover-marker path later lands (a second consumer), confirm the arithmetic was promoted to a `gate-ledger` read verb, not copy-pasted into a second prose site |
